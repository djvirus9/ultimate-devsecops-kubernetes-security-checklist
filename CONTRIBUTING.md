## Contributing

Thanks for helping improve this repository. Keep changes focused and easy to review.

### Workflow

1. Create a branch from `main`.
2. Keep commits small and descriptive.
3. Update docs/labs/templates when behavior changes.
4. Open a pull request and complete the checklist.

### Pull request checklist

- [ ] Summary explains what changed and why
- [ ] Docs or labs updated when applicable
- [ ] Tests or validation steps noted
- [ ] Security impact reviewed and documented

### Code of conduct

Be respectful and constructive.
